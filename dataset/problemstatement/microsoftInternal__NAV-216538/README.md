Title: [master] [All-E] Price is not updated correctly based on Price Lists in Service Objects when Quantity is Changed
Repro Steps:
The price is not updated in the subscription when the quantity is changed Repro steps 26.x 1) Setup the following data: Item / Item Variants / Sales Price Liste 2) Create a Sales Quote and check the unit prices according to «Variant Code» and «Quantity» => OK 3) Create a Subscription for Item 1000 / Variant MONTH / Quantity 15 and check the unit price => 240 OK 4) Change Subscription from Variant MONTH to YEAR and check the unit price => 140 OK 5) Change Subscription from Quantity 15 to 25 and check the unit price => 140 WRONG Although the message indicates that changing the quantity will update the price, the price remains unchanged. The price is only updated correctly, if the item variant is changed. ---------------------------- Steps to reproduce in BC25.5 (CH-Version) without any customizations Enable «New sales pricing experience» Create a new «Service Commitment Template» Create a new «Service Commitment Package» and select the newly created «Service Comitment Template» Create a new «Item» Add the «Service Commitment Package» to the newly created «Item» Add two «Item Variants» to the newly created «Item» Open the default «Sales Price List» and add the following prices Create an new «Customer Contract» for Adatum Corporation Run action «Extend Contract» and add one piece of item 1000 Since there is no «Price» in the newly created contract line, click on the «Quantity» to open the «Service Object» On the «Service Object», add the field «Variant Code» via action «Personalize». Select «Item Variant» MONTH and accept the following message Check the «Calculation Base Amount» => this has changed to 250, which is correct Change the «Variant Code» from MONTH to YEAR and accept the following message Check the «Calculation Base Amount» => this has changed to 150, which is correct Change the «Quantity» from 1 to 25 and accept the following message Check the «Calculation Base Amount» => this has NOT changed, which is wrong => it should be 130 EXPECTED RESULT: Calculation Base Amount should be 130 ACTUAL RESULT: Calculation Base Amount remains 150
Description:
If the quantity is changed in a service object, a message appears stating that the price will be updated, but this is not carried out. The prices are not changed, which is incorrect. If, on the other hand, the “Variant Code” is changed in a service object, the message is also displayed and the price is actually updated, which is correct.

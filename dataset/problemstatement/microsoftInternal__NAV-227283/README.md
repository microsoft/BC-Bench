Title: The changes to the Gen. Journal Line record cannot be saved because some information on the page is not up-to-date. Close the page, reopen it, and try again.
Repro Steps:
Open a brand new Cronus environment. Go to General Ledger Setup. Appln. Rounding Precision 0.05. Pmt. Disc. Tolerance Warning Set to true. Payment Discount Grace Period Set to 5D. Payment Tolerance Warning Set to true. Open Cash Receipt Journal and choose Customer 1000. Click Apply Entries to check the invoice Pmt Discount Date and Pmt Discount Tolerance Date. Go Back and change the posting date and choose a date between Pmt Discount Date and Pmt Discount Tolerance Date. for example 8/17/2025. Now Choose Invoice in Applies to Doc Type and move to Applies to Doc No. Click Ok. Change the Amount to less Amount. Once you change the Amount, a pop up window will appear , choose the first option. Once done, click on the second empty line to fill it, the error will occur. Error " The changes to the Gen. Journal Line record cannot be saved because some information on the page is not up-to-date. Close the page, reopen it, and try again. Identification fields and values: Journal Template Name='XXX',Journal Batch Name='XXX',Line No.='10000' Expected Results: The error shouldn't occur and the user is supposed to be able to add values to the second line.

Title: [master] [ALL-E] Line & Invoice Discounts disappears once you re-open Sales Order to add a Campaign no.
Repro Steps:
1- Open Business Central 24.0. 2- Search for Sales orders. 3- Create a new sales order. 4- Add the following lines with lines discounts and invoice discount. 5-Make sure the campaign no is empty and release. 6- Now try to re-open the document once more to add a campaign no. 7- As you can see the line discounts and the invoice discounts disappeared. 8- When you try to release the document once more, the following error occurs. " The changes to the Sales Header record cannot be saved because some information on the page is not up-to-date. Close the page, reopen it, and try again. Identification fields and values: Document Type='Order',No.='101011'" ​ 9- To avoid the error message, you have to apply the same values again for line discounts and invoice discounts before releasing the document. Actual Results: The line discounts and invoice discounts disappears once you re-open the sales order to add a campaign no. Expected Results: The line discounts and invoice discounts should not disappear once you re-open the sales order to add a campaign no. Dusan Jakovljevic's edit: Please check Andrei's and Ivan's comment. We should add notification that prices/discounts might have changed (similar to other parameters as Andrei mentioned) and remove this error. Workaround: The customer adds the values manually to avoid the message.

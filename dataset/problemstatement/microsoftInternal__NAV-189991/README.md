Title: [master]  The No. Printed Field (47 Integer) is automatically set to 1 in the Posted Sales Invoice after posting the Sales Order (The New Digital Voucher Functionality is enabled)
Repro Steps:
Issue was reproduced in Version: GB Business Central 24.2 (Platform 24.0.21096.0 + Application 24.2.20227.20460) Enable the Digital Voucher Functionality 2. Create a new Sales Order and Post the Sales Order 3. Ship and Invoice 4. Open the Posted Sales Invoice and we would see that the Voucher is automatically added, which is fine 5. When we zoom in on the page, we see that the No. Printed Field is set to 1 automatically despite the Posted Sales Invoice not printed yet. ================================================================= ACTUAL RESULT: The No. Printed Field (47, Integer) is set to 1 automatically on the Posted Sales Invoice despite not being printed yet ================================================================== Expected RESULT: The No. Printed Field (47, Integer) should be set to 0 as the end user uses the field to identify how many times they have printed the Posted Sales Invoice.
Description:
The No. Printed Field (47 Integer) is automatically set to 1 in the Posted Sales Invoice after posting the Sales Order (The New Digital Voucher Functionality is enabled)

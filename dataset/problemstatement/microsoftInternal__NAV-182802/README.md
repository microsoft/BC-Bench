Title: [ALL-E] Automatic reference to the first price list instead of reassigning a new number series.
Repro Steps:
The issue is tested on W1, AT & DE Activate new sales price experience in feature management page. Open either sales or purchase price list. Type in anything in the "Code" field. Clean what you typed in the "Code" field and then click on "Tab" key on the keyboard. The first price list will be opened instead of assigning a new number series. Actual result: The next number series is not fetched, instead, the system opens the first price list. If you do the same without writing and deleting, then the next number in the serials is taken as expected. Expected result: The next number series should be fetched and assigned instead of opening already existing price list.
Description:
Automatic reference to the first price list instead of reassigning a new number series.

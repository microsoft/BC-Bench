Title: [ALL-E] SKU card created for a Non-Inventory type of item has the Planning and Warehouse tabs, unlike the main item card.
Repro Steps:
Create a new Item and set the 'Type' = "Non-Inventory" Review the item card and notice we don't have a 'Planning' tab or 'Warehouse' tab...which is by design. But then go create a Stockkeeping Unit for this item (Actions > Functions > Create Stockkeeping Unit for BLUE location. Now go into the SKU and see you have a Planning Tab. Customer is complaining that they are not getting Planning suggestions for this Non-Inventory Type of item, even though they created a SKU and can set a Reorder Policy.This is just an oversight and we should not have Planning tab or warehouse tab for SKU of a Non-Inventory Type of item..."Service" type as well for that matter.
Description:
(Collab with Juan who is assisting case owner Balal Nazir...)Non-Inventory Type of item card does Planning Tab. By design, these types of item cannot be used for planning.

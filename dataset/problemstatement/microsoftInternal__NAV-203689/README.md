Title: [master] [ALL-E] Invoice Discounts are validated automatically to sales order when user changes posting date
Repro Steps:
Invoice Discounts are validated automatically to sales order when user changes posting date and Calculate Invoice Discount is off on Sales and Receivables setup. Issue was reproduced in Version: GB Business Central 25.1 (Platform 25.0.26191.0 + Application 25.1.25873.25976) Ensure that Calculate Invoice Discount is switched off on the Sales & Receivables setup On Customer Card, navigate to Related Prices and Discounts -> Invoice Discounts Customer Posting Group set a service Charge Account. Create a new Sales Order, add Athens Desk Change the Posting Date and the Service charge is calculated on the Sales Order. =============Actual Result=================================== When you change the posting Date on the first Order, The Service Charge is calculated automatically. ===============Expected Result=============================== When you change the posting Date on the Sales Order, the Service Charge should not be calculated automatically. Calculate Invoice Discount is off on Sales and Receivables setup
Description:
Invoice Discounts are validated automatically to sales order when user changes posting date

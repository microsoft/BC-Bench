Title: [master] [ALL-E] Field Drop Shipment is not filled correctly when using Copy Document functionality
Repro Steps:
Open a W1 23.5 database First Update "Purchasing Code" for the items as shown below, Item 2000-S- Drop Shipment Create a Sales Quote for a Customer - 10000 with 1x item 2000-S Release the Sales Quote. Create a Blanket Sales Order. Select the Customer- 10000 Click "Prepare" - Copy Document Option. In Copy Sales Document window, select below values, Document Type = Quote Select Document No from the list (Sales quote Document No) Enable Include Header option. Click Ok In Blanket Sales Order, In "Lines Section" select line with item 2000-S and open page Inspection. Search for "Drop Shipment" in page Inspection Actual Result: Drop Shipment is No for the line. Extepced Result: Drop shipment should be Yes More Information: Same happens when you use Purchasing Code SPEC ORDER. Then field Special Order or the line is No. When you manually add the item to the line, then field Drop Shipment for Purchasing Code Drop Shipment is Yes. Same for SPEC ORDER - when you manually add the item to the line, field Special Order is Yes.
Description:
When you create a document with "Copy Document" functionality and the used item has a purchase code setup, not all fields are validated as expected.

Title: [master] [ALL-E] Account ranges are not getting recognized when the Totaling Type is Statistical Account in Financial reports row definition
Repro Steps:
1. Create three new Stat Accounts 2. Enter stat account journals for all the 3 accounts and register them. 3. Verify that the entries are registered. 4. Now create a new Row Definition Setup the row definition to total up the 3 stat accounts as follows: Totalling Type = Statistical Accounts Totalling = 10000..30000 6. Create a new Financial Report and choose the "STAT ROW" for row definition and "DEFAULT" for column definition. 7. Run the Report. Make sure the correct period is specified. Notice that the net change shows only 1000. Wheres as that line should be 3000 since we etered 1000 each for for all 3 stat accounts and that row definition is configured to show the total of stat accounts 10000..30000. 8. Further, when you drill down on the Netchange field, you would notice that the system is setting the filter on the first account only and not considering the range speficied on the row definition. Current Result: Although we have defined a formula in the totaling column, we only see the value of the first statistical account. Expected Result: We should see the values from all 3 Statistical Accounts that are defined in the totaling column. In this case the amount should be 1000.
Description:
Account ranges are not getting recognized when the Totaling Type is Statistical Account in Financial reports row definition
